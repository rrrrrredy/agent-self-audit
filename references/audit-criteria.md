# 自检判断标准详细说明

## 检查项一：Skills 数量

| 范围 | 状态 | 建议 |
|------|------|------|
| < 50 | ✅ 健康 | 无需操作 |
| 50-70 | 🟡 偏多 | 检查是否有无用 skill |
| > 70 | 🔴 过多 | 立即清理 |

**识别无用 skill 的标准**：
- macOS 专属工具（apple-notes、bear-notes 等）
- 智能家居（openhue、sonoscli 等）
- 欧美平台（discord、slack、notion 等）
- 依赖不存在的 API key

## 检查项二：MEMORY.md 长度

| 行数 | 状态 | 建议 |
|------|------|------|
| < 80 | ✅ 健康 | 无需操作 |
| 80-120 | 🟡 偏长 | 建议精简 |
| > 120 | 🔴 过长 | 必须精简 |

**精简优先级**（从高到低）：
1. 已有对应 facts.yaml 条目的内容 → 删除
2. 超过 30 天未引用的规则 → 删除或迁移
3. 重复/相似条目 → 合并
4. 细节性内容 → 迁移到 facts.yaml

## 检查项三：Cron 蒸馏机制

完整的蒸馏 prompt 必须包含：
- 新事实 → facts.yaml
- MEMORY.md 精简（硬性行数约束 ≤80 行）
- 旧日志归档或清理

## 检查项八：facts.yaml 健康度

| 条件 | 状态 |
|------|------|
| 文件不存在 | 🔴 关键缺失 |
| 行数 > 500 | 🟡 偏长 |
| 行数 < 10 | 🟡 记录过少 |
| 最近修改 > 30 天 | 🟡 长期未维护 |
| 正常 | ✅ 健康 |
